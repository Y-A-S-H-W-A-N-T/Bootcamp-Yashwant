Managing Environment Variables (dotenv)

Use the dotenv library to manage configuration via environment variables.
Level 1

Problem Description: 1. Install dotenv and create a .env file with configuration variables (e.g., PORT, DB_HOST). 2. Load the variables into the program and use them for configuration.

Level 2

Problem Description: 1. Add fallback values for missing environment variables. 2. Log a warning when a variable is missing.

Level 3

Problem Description: 1. Use environment variables to configure a database connection and API credentials. (You can use sqlite file based database for this exercise.) 2. Securely log sensitive configuration details.
